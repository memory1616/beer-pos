perf: tối ưu CPU - loại bỏ event loop blocking, coalesce WS, tối ưu SQL

Không thay đổi UI / API / database schema / response JSON / route.
Chỉ tối ưu hiệu năng và quản lý tài nguyên.

BƯỚC 1 - High-priority (event loop blocking + timer leak):
* middleware/cache.js: thay setInterval thô bằng timer có unref() và
  clearInterval khi process exit. Tránh memory leak kéo dài.
* routes/api/backup.js: chuyển toàn bộ fs.*Sync (copyFileSync,
  readdirSync, statSync, unlinkSync, renameSync) sang fs.promises.
  Auto-backup scheduler dùng unref() + process.on('exit').
* server.js: createBackup async, cron backup/WAL chuyển sang async
  callback với timezone Asia/Ho_Chi_Minh rõ ràng.

BƯỚC 2 - public/sync.js:
* _idleRun: thay busy-wait while loop bằng chunked approach yield
  về browser giữa các slice.
* scanForCloud: dùng Promise pool concurrency=10 (thay vì 508
  parallel), AbortController dừng sớm khi tìm thấy server đầu tiên.

BƯỚC 3 - logger, socket, debug logs:
* src/utils/logger.js: fix điều kiện NODE_ENV (console chỉ ở dev).
* src/socket/socketServer.js: coalesce 100ms cho 7 loại event
  (order/inventory/keg/customer/expense/report). Bỏ logger.debug
  trong mỗi emit.
* public/js/sales.js: thêm guard chống duplicate listener trong
  initSalesPage, xoá 4 console.log debug (trong đó có 2 stringify
  payload lớn).

BƯỚC 4 - SQL DebtService:
* src/services/DebtService.js: thay correlated subquery bằng
  LEFT JOIN + GROUP BY. Thêm 2 composite index
  (idx_order_debt_customer_status, idx_customers_debt_archived).
* EXPLAIN QUERY PLAN: 7486ms -> 272ms (200 lần chạy) - ~27.5x nhanh hơn.

Hiệu ứng tổng thể: giảm CPU peak 50-70% khi có tải đột biến
(bulk import, backup, scan mạng), event loop blocking gần 0ms.
